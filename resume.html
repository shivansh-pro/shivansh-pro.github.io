Topic Knowledge & Application
Medallion Architecture Worked in a layered Raw/Conformed pipeline architecture in production – the direct operational predecessor of Bronze/Silver in Medallion. Currently studying the full Bronze/Silver/Gold pattern through DP-600 coursework and hands-on Microsoft Fabric work. Key principle: each layer has a distinct data contract – immutable raw ingest, validated/conformed records, and analytics-ready aggregates.
OLAP Cubes vs. Lakehouse OLAP cubes (SSAS, Essbase) are pre-aggregated multidimensional structures – rigid schema, expensive maintenance, poor support for ad-hoc queries and real-time data. Modern data lakehouses replace cubes with on-demand query engines (Synapse Serverless SQL, Fabric SQL Endpoint, Spark) against Parquet/Delta files – flexible schema, live data, and and Direct Lake mode for sub-second BI without pre-aggregation.
Star Schema / Dimensional Modeling A dimensional modeling pattern: central fact table (measurable events) surrounded by dimension tables (descriptive context). Queries are simple and fast because joins are predictable and shallow. Study-level knowledge being applied through DP-600 and Fabric Lakehouse design – relevant to designing the Gold layer that feeds Power BI semantic models in enterprise real estate platforms.
SCD Type 2 A pattern for tracking historical changes to dimension attributes over time. When a dimension record changes, SCD Type 2 expires the old row and inserts a new row with updated values and a new effective date – preserving full history. In Delta Lake, implemented via MERGE INTO with effective date and is_current flag logic. Currently building through DP-600 study – directly relevant to lease history and tenant record tracking use cases in enterprise property management platforms.
Dimensional Models for Self-Service Analytics A Gold layer data structure designed so business analysts can build their own reports in Power BI without needing a data engineer to write SQL for every new question. Key design principles: intuitive naming, pre-joined structures, consistent grain, and documented measures. Study-level knowledge being built through DP-600.
Databricks vs. Fabric Spark Both run Apache Spark on Delta Lake. Databricks = independent platform with Unity Catalog governance, MLflow, and strong ML/DS tooling. Microsoft Fabric = integrated Microsoft ecosystem (Power BI, Purview, ADF, Synapse) with OneLake as unified storage. Fabric preferred when deep Power BI + Azure AD integration matters; Databricks preferred for ML-heavy workloads or multi-cloud.
Snowflake vs. Synapse vs. Fabric Snowflake = cloud-agnostic, fully managed SQL warehouse with strong separation of storage and compute. Azure Synapse = Microsoft-native, integrates SQL + Spark in one platform. Microsoft Fabric = newest, unifies Synapse + Power BI + ADF + ML under one license and OneLake. For Azure-native stacks, Fabric Warehouse supersedes standalone Synapse Dedicated.
